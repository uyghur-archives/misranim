كومپىيوتېر بىلىملىرىدىن سۇئال-جاۋابلار

CPU ئالماشتۇرۇش كارتىسى ئارقىلىق ئاساسىي تاختىغا ئۇلىنىدۇ. يېقىندىن بېرى كومپىيوتېرنى ئاچقاندا ئېكراندا ھېچقانداق ئۇچۇر يوق. توك مەنبەسىنى ئالماشتۇرغاندىن كېيىنمۇ ئەھۋال يەنىلا ئوخشاش، بۇ قانداق ئەھۋال؟
ج: ئېكراندا ھېچقانداق ئۇچۇر كۆرۈنمەسلىكى توك مەنبەسى، ئاساسىي تاختا، CPU قاتارلىقلارنىڭ كاشىلىسى. ئالماشتۇرۇش كارتىسى ئىشلىتىلگەن كومپىيوتېرغا نىسبەتەن بۇ چوقۇم ئالماشتۇرۇش كارتىسىدا كۆرۈلگەن مەسىلە. CPU نىڭ توك سەرپىياتى ناھايىتى چوڭ بولغاچقا ئۇزۇن مەزگىل ئىشلەتكەندىن كېيىن، ئالماشتۇرۇش كارتىسىنىڭ ئاساسىي تاختىغا تۇتىشىدىغان قىسمى ئوكسىدلىنىپ ياخشى تۇتاشمىغانلىقتىن كومپيوتېر نورمال ئىشلىمەيدۇ. بۇ مەسىلىنى ھەل قىلىش ئۈچۈن ئاساسىي ماشىنىنى ئېچىپ، ئالماشتۇرۇش كارتىسىنى ئېلىپ سۇسىز ئىسپىرىت بىلەن ئالماشتۇرۇش كارتىسىنىڭ ئاساسىي تاختا بىلەن تۇتىشىپ تۇرىدىغان قىسمىنى پاكىزە تازىلاپ قايتا ئورۇنلاشتۇرغاندىن كېيىن مەسىلە ھەل بولىدۇ.
س: كومپىيوتېرنى قوزغاتقاندا CPU توغرا ئەمەس دىگەن خەت چىقىۋالىدۇ، بۇ قانداق ئەھۋال؟ CMOSقا كىرگەندە كومپىيوتېر قېتىۋالىدۇ. ئوخشاش تىپتىكى CPU نى ئالماشتۇرساق نورمال ئىشلەيدۇ، بۇنى قانداق ھەل قىلىش كېرەك؟
ج: بىر قىسىم 815EP ئاساسىي تاختىنىڭ Manage Easy ئىقتىدارى بولۇپ،  قاتتىق دىسكىنىڭ نورمال ياكى نورمال ئەمەسلىكىنى تەكشۈرۈپ تۇرىدۇ. CPU توغرا ئەمەسلىك CPU نىڭ ئۆزىدىكى خاتالىق ئەمەس. CMOS قا كىرىپ PC Health Status تۈرىنى تاللاپ CPU نىڭ تېمپراتورىسى، شامالدۇرغۇچنىڭ ئايلىنىشىنى تەكشۈرگەندە CPU نىڭ تېمپراتورىسى بەكلا يوقىرى(90~80 گرادۇس)بولسا ، چوقۇم شامالدۇرغۇچنىڭ كاشىلىسى. شامالدۇرغۇچنى ئالماشتۇرغاندىن كېيىن مەسىلە ھەل بولىدۇ. ئەگەر يەنىلا مەسىلە ھەل بولمىساCPU ياكى ئاساسىي تاختىنىڭ ئۆزىدىكى ئېغىر كاشىلا. بۇنى ساتقان ئورۇنغا ياكى كەسپى رېمونتخانىغا ئاپىرىڭ.
س: CPU نىڭ چاستوتىسىنى تۆۋەنلىتىش زىيانلىقمۇ؟
ج: CPU نىڭ چاستوتىسىنى تۆۋەنلىتىش زىيانلىق ئەمەس. بىراق بۇ مۇتلەق ئەمەس. ئۆلچەملىك سىرتقى چاستوتىنى ئىشلەتكەندە ئاساسىي تاختىدىكى PCI باش لىنىيە چاستوتىسىمۇ ئاپتوماتىك تەڭشىلىدۇ. ئۆلچەمسىز سىرتقى چاستوتىنى ئىشلەتكەندە مەسىلەن : CPU سىرتقى چاستوتىسى 133MHz بولسا، بۇنى 112MHz قىلىپ ئىشلەتكەندە ئاساسىي تاختىنىڭ PCI باش لىنىيە نورمال بولمىغان ھادىسە كېلىپ چىقىدۇ. ھەم كومپيوتېر نورمال ئىشلىمەسلىك بىر قىسىم قاتتىق دېتاللارنىڭ بۇزۇلۇشىنى كەلتۈرۈپ چىقىرىدۇ. شۇڭا CPU چاستوتى ئۆلچەملىك چاستوتىدا ئىشلىگىنى ئەڭ مۇۋاپىق .
س: CPU نىڭ توك بېسىمىنى تۆۋەنلىتىش ئارقىلىق تېمپراتورىسىنى تۆۋەنلىتىشكە بولامدۇ؟
ج: بولىدۇ. ئوخشاش بولمىغان تىپتىكى ۋە ئوخشاش بولمىغان ماركىدىكى ئاساسىي تاختىلارغا نىسبەتەن CPU نىڭ توك بېسىمىنى تۆۋەنلىتىش ئۇسۇللىرى ئوخشاش بولمايدۇ، شۇنىڭ ئۈچۈن ئاساسىي تاختىنىڭ چۈشەندۈرۈش ماتېرىيالىغا ئاساسەن دىققەت بىلەن ئىش كۆرۈڭ.  خاتا تەڭشەش سەۋەبىدىن CPU نىڭ بۇزۇلۇپ كېتىشىدىن ساقلىنىڭ.
س: CPU نىڭ بىر « پۇتى» سۇنۇپ كەتسە قانداق قىلىش كېرەك؟
ج: سۇنۇپ كەتكەن پۇتنى ئۆز ئورنىغا قويۇپ بىر تامچە 502 يىلىمىنى تېمىتقاندىن كېيىن ئۈچ مىنۇت ئىچىدە مۇقىملىشىدۇ. ياخشى تۇتىشىپ بولغاندىن كېيىن، ئاساسىي تاختىغا مۇقىملاشتۇرۇپ ئىشلەتسىڭىز بولىدۇ.
س: CPU تۈرىدىكى AMD دا ماسلاشماسلىق مەسىلىسى مەۋجۇتمۇ ؟
ج: CPU تۈرىدىكى AMD دا ماسلاشماسلىق مەسىلىسى مەۋجۇت ئەمەس. CPU تۈرىدىكى Intel دا ئىجرا بولغان بارلىق پروگراممىلار CPU تۈرىدىكى AMD دىمۇ ئىجرا بولىۋېرىدۇ. ماسلاشماسلىق مۇقىمسىزلىق ئەھۋاللىرى بولسا ئاساسىي تاختىدىكى مەسىلە بولۇپ، بۇنى ئەڭ يېڭى قوزغىتىش پروگراممىسى ياكى تولۇقلىما پروگرامما ئارقىلىق ھەم شۇنداقلا BIOS نەشىرىنى يېڭىلاش ئارقىلىق ھەل قىلغىلى بولىدۇ.
س: CPU نىڭ چاستوتىسىنى يوقىرى كۆتۈرۈش ئارقىلىق سىستېمىنىڭ تېزلىكىنى ئاشۇرغىلى بولامدۇ؟ ھەر قانداق CPU نىڭ چاستوتىسىنى يوقىرى كۆتۈرگىلى بولامدۇ؟CPU نىڭ چاستوتىسىنى  يوقىرى كۆتۈرۈش CPU نىڭ ئۆمرىگە تەسىر يەتكۈزەمدۇ؟
ج: CPU نىڭ چاستوتىسىنى يوقىرى كۆتۈرۈش ئارقىلىق تېزلىكنى ئاشۇرغىلى بولىدۇ. ھازىرقى CPU لارنىڭ ھەسسىلىك سانى تاقىۋېتىلگەن بولۇپ، سىرتقى چاستوتىسى ئارقىلىق چاستوتىسى يوقىرى كۆتۈرۈلىدۇ. ھەر قانداق CPU نىڭ چاستوتىسىنى يوقىرى كۆتۈرگىلى بولمايدۇ. سىستېمىنىڭ تېزلىكى ھەر قايسى قاتتىق دېتاللار بىلەن زىچ مۇناسىۋەتلىك بولغاچقا CPU نىڭ چاستوتىسىنى يوقىرى كۆتۈرگەن بىلەن سىستېمىنىڭ تېزلىكى ماس ھالدا يوقىرى كۆتۈرلۈشى ناتايىن.
CPU نىڭ چاستوتىسىنى يۇقىرى كۆتۈرگەندىن كېيىن بەك قىزىيدۇ. مۇقىمسىزلىشىدۇ. CPU نىڭ ئۆمرىگىمۇ تەسىر يەتكۈزىدۇ، ھەتتا كۆيۈپ كېتىدۇ، ئەڭ ياخشىسى CPU نىڭ چاستوتىسىنى يوقىرى كۆتۈرمەي ھەم تۆۋەنلەتمەي، نورمال چاستوتىدا ئىشلەتكەن ياخشى .
س: CPU چاستوتىسىنى يوقىرى كۆتۈرگەندە تىزىملاش جەدۋىلىنىڭ بۇزۇلۇش ھادىسىسى قانداق ئىش؟
ج: بۇ CPU نىڭ چاستوتىسىنى يوقىرى كۆتۈرۈش بىلەن مۇناسىۋەتلىك. CPU چاستوتىسىنى يوقىرى كۆتۈرگەندىن كېيىن سىرتقى چاستوتىمۇ يوقىرى كۆتۈرۈلىدۇ. ماس ھالدا  ئىچكى ساقلىغۇچنىڭمۇ چاستوتىسى يوقىرى كۆتۈرۈلىدۇ. ئەگەر ئىچكى ساقلىغۇچنىڭ چاستوتىسى 100MHz بولسا، ئۇچۇر يەتكۈزۈشتە خاتالىق پەيدا بولۇپ، تىزىملاش جەدۋىلىنىڭ بۇزۇلۇشىنى كەلتۈرۈپ چىقىرىدۇ.
CPU نىڭ چاستوتىسىنى ئەسلىگە كەلتۈرۈش ياكى مۇۋاپىق چاستوتىدىكى ئىچكى ساقلىغۇچنى ئالماشتۇرۇش ئارقىلىق بۇ مەسىلىنى ھەل قىلىشقا بولىدۇ. 
س: BIOS دىن CPU نىڭ تېمپراتورىسىنى تەكشۈرگەندە 95 گرادۇس، شامالدۇرغۇچنىڭ ئايلىنىش سۈرئىتى 0 بولغان، بۇ قانداق ئەھۋال؟
ج: BIOS نەشىرى تۆۋەن بولىشى مۇمكىن، ھەم شامالدۇرغۇچنىڭ سۈپىتى ياخشى ئەمەس. شامالدۇرغۇچنىڭ ئايلىنىش سۈرئىتىنىڭ 0 بولىشى شامالچى ئۆلچەملىك بولمىغاچقا ئاساسىي تاختا تەكشۈرەلمىگەن ھەم شامالدۇرغۇچ توك مەنبەسى ئاساسىي تاختىدىكى CPU Fan روزىتكىغا ئۇلانمىغان. بۇ مەسىلىنى ھەل قىلىش ئۈچۈن :
1. BIOS نەشىرىنى يوقىرى كۆتۈرۈڭ. سۈپىتى ياخشى ئۆلچەملىك شامالدۇرغۇچ ئىشلىتىڭ.
2. شامالدۇرغۇچنىڭ توك مەنبەسىنى چوقۇم ئاساسىي تاختىدىكى CPU Fan روزىتكىسىغا ئۇلاڭ.
س: CPU چاستوتىسىنى يوقىرى كۆتۈرگەندىن كېيىن نورمال. بىراق 3D ئويۇنلىرىنى ئوينىغاندا كومپيوتېر قېتىۋالىدۇ، بۇ قانداق ئەھۋال؟
ج: 3D ئويۇنلىرىنىڭ CPU غا بولغان تەلىپى يوقىرى بولغاچقا، 3D ئويۇنلىرىنى ئوينىغاندا CPU بەكرەك قىزىيدۇ. شۇ سەۋەبتىن كومپيوتېر قېتىۋالغان. ئەڭ ياخشىسى CPU چاستوتىسىنى يوقىرى كۆتۈرمەك. ئەگەر يوقىرى كۆتۈرۈش زۆرۈر بولسا، سۈپىتى ياخشى شامالچىغا ئالماشتۇرۇڭ ھەم ئىسسىقلىقنىڭ تولۇق تارقىلىشىغا كاپالەتلىك قىلىڭ.
س: بىر نەچچە پۇتى سۇنۇپ كەتكەن CPU نى ئىشلىتىشكە بولامدۇ؟
ج: CPU نىڭ پۇتلىرى ئىنتايىن ئىنچىكە ھەم زىچ بولغاچقا، ئادەتتە قول بىلەن كەپشەرلەش ئىنتايىن قىيىن. كەپشەرلىمەيمۇ ئىشلىتىشكە بولىدۇ. 
1. توك ئۆتكۈزۈش ئىقتىدارى ياخشى . ئاسان داتلاشمايدىغان ئىنچىكە سىم تەييارلاپ CPU نىڭ ئەسلىدىكى پۇتى بىلەن تەڭ قىلىپ كېسىمىز.
2. ئالدىن تەييارلىۋالغان سىمنى CPU نىڭ ئۈزۈلگەن پۇتى چۈشىدىغان ئورۇنغا سېلىپ ، CPU نى ئورۇنلاشتۇرىمىز.
3. قىسقۇچنىڭ ئېلاستىكلىقى ياخشى بولغان CPU شامالدۇرغۇچىنى ئورۇنلاشتۇرساق كومپيوتېر نورمال ئىشلەيدۇ. ئەگەر نورمال قوزغالمىسا قايتا سىناڭ. چوقۇم مۇۋاپپەقىيەتلىك بولىدۇ. 
س: كومپيوتېر ئەسلىدە  نورمال ئىدى، بىر نەچچە كۈن ئىشلەتمەي قوزغاتقاندىن كېيىن، ئېكراندا ھېچقانداق ئۇچۇر يوق، بۇ قانداق بولغىنى ؟
ج: بۇ خىل كاشىلا ئاساسلىقى CPU نىڭ ياخشى تۇتاشماسلىقىدىن بولغان. كومپيوتېر نورمال ئىشلىگەندە CPU نىڭ توك سەرپىياتى يوقىرى ھەم بەك قىزىيدىغان بولغاچقا CPU پۇتلىرى ئوكسىدلىنىش سەۋەبىدىن ياخشى تىگىشمەيدىغان ئەھۋال شەكىللىنىدۇ. شۇڭا سىز ئاساسىي ماشىنا قېپىنى ئېچىپ CPUنى چىقىرىپ، چوتكىسى بىلەن CPU نىڭ پۇتلىرىنى تازىلاڭ ھەم CPU روزىتكىسىدىكى چاڭ توزانلارنىمۇ تازىلاپ، CPU نى قايتىدىن ئورۇنلاشتۇرسىڭىز، كومپيوتېر نورمال قوزغىلىدۇ.                                                                                                                             
س: كومپيوتېرنى مەلۇم ۋاقىت ئىشلەتكەندىن كېيىنCPU شامالچىسىدىن ئاۋاز چىقىپ قېتىۋالىدۇ ياكى قايتا قوزغىلىپ كېتىدۇ، بۇ قانداق بولغىنى؟
ج: بۇ خىل ھادىسە CPU شامالچىسىغا كۆپ مىقداردا چاڭ توزان توپلىشىۋالغانلىقى سەۋەبىدىن بولغان. چاڭ - توزان سەۋەبىدىنCPU شامالچىسىدا ئاۋاز پەيدا بولغان ھەم ئايلىنىش نورمال نورمالسىزلىقى تۈپەيلىدىن ئىسسىقلىق تارقىتىش ياخشى بولمىغاچقا، CPU قىززىپ ، كومپيوتېر قېتىۋالغان، CPU شامالچىسىدىكى چاڭ - توزاننى تازىلىۋېتىڭ، ياكى شامالچىنى ئالماشتۇرۋېتىڭ.
س: كومپيوتېرنى قوزغىتىپ ئۇزاق ئۆتمەي قايتا قوزغىلىدۇ، يەنە قايتا قوزغالغاندىن كېيىن قوزغىلىپلا قايتا قوزغىلىدۇ ياكى قوزغالمايدۇ، بۇ قانداق ئەھۋال؟
ج: كومپيوتېرغا ۋىرۇس يۇققان بولىشى مۇمكىن. يېڭى نەشىردىكى ۋىرۇس تازىلاش يۇمشاق دېتالى بىلەن ۋىرۇس تازىلاڭ. ئۇنىڭدىن باشقا ھاۋا تېمپراتورىسى بەك يوقىرى بولىشى ياكى CPU شامالدۇرغۇچىنىڭ سوۋۇتۇش ئىقتىدارى ياخشى بولمىغانلىقتىن بولغان. سوۋۋۇتۇش ئىقتىدارى ياخشى بولغان شامالچىغا ئالماشتۇرۇڭ .

ج: شامالچە توك مەنبەسى ئۈچ تال ياكى ئىككى تال بولىدۇ. توك مەنبە سىمى ئۈچ تال بولغان شامالچىنىڭ ئايلىنىش سۈرئىتىنى BIOS ئۆلچىيەلەيدۇ. ئىككى تال بولغاننى ئۆلچىيەلمەيدۇ. ئۇنىڭدىن باشقا بىر قىسم سۈپىتى ياخشى بولمىغان شامالچىلاردىمۇ يوقارقىدەك ئەھۋال مەۋجۇت. شۇڭا سۈپىتى ياخشى شامالچى سەپلەڭ.
س: ئىچكى ساقلىغۇچنىڭ كونا ياكى يېڭىلىقىغا قانداق ھۆكۈم قىلىمىز؟
ج: ئىچكى ساقلىغۇچ ئۆزەك پلاستىنكىسى ئۈستىدە 9811 غا ئوخشاش خەت شەكىللىرى بولىدۇ. بۇ مۇشۇ ئىچكى ساقلىغۇچنىڭ 1998-يىلىنىڭ 11- ھەپتىسى زاۋۇتتىن چىققانلىقىنى بىلدۈرىدۇ. بۇنىڭدىن باشقا كالېندار ئارقىلىق RAM نىڭ كونا يىڭىلىقىغا ھۆكۈم قىلغىلى بولىدۇ.
س: ئىچكى ساقلىغۇچنىڭ يىتىشمەسلىكى قانداق ئەھۋال؟
ج: كومپيوتېر سۈرئىتى ئاستىلاپ كەتكەندە قايتا قوزغاتقاندىن كېيىن، ئويۇنچۇق ئوينىغاندا ئىچكى ساقلىغۇچ يېتىشمەسلىك ئۇچۇرى كۆرىنىدۇ. ھەل قىلىش ئۇسۇللىرى :
1. كومپيوتېرنى ئۆچۈرۈپ قايتىدىن قوزغىتىڭ. ئىچكى ساقلىغۇچتىن بىرنى قوشۇڭ.
2. ئىچكى ساقلىغۇچتا توقۇنۇش بولسا، باشقا ئىچكى ساقلىغۇچقا ئالماشتۇرۇڭ.
3. ئىچكى ساقلىغۇچتا سۈپەت مەسىلىسى بار ياكى بۇزۇلغان.
س: ئىچكى ساقلىغۇچ چاستوتىسىنى يوقىرى كۆتۈرگەندىن كېيىن كومپيوتېر قوزغالمايدىغان، قايتا قوزغىلىپ كېتىدىغان ۋە قېتىۋالىدىغان ئەھۋاللارنى قانداق ھەل قىلىش كېرەك؟
ج: 1.ئادەتتە ئىچكى ساقلىغۇچ چاستوتىسى يوقىرى كۆتۈرۈلۈپ كەتسە سانلىق مەلۇماتلارنى ساقلاش - ئوقۇش جەريانىدا ئاسانلا خاتالىق كۆرۈلۈپ، يوقارقىدەك ئەھۋال كۆرىلىدۇ. ئىچكى ساقلىغۇچ چاستوتىسىنى يوقىرى كۆتۈرگەندىن كېيىن كومپيوتېر نورمال قوزغالمىسا CMOS ئۇچۇرلىرىنى ئۆچىرىدىغان سىم ئاتلاتقۇچ ياكى CMOS باتارىيىسىنى ئېلىۋېتىش ئارقىلىق CMOS ئۇچۇرلىرىنى ئۆچۈرۈۋېتىپ، كومپيوتېرنى قايتا قوزغاتقاندىن كېيىن ئىچكى ساقلىغۇچ چاستوتىسىنى مۇۋاپىق تەڭشەڭ ياكى ئەسلىدىكى چاستوتىغا تەڭشەڭ.
2. كومپيوتېر قوزغىلىپ نورمال ئىشلىمىسە، ئوخشاشلا CMOS دىن ئىچكى ساقلىغۇچ چاستوتىسىنى مۇۋاپىق تەڭشەڭ ياكى ئەسلىدىكى چاستوتىغا تەڭشەڭ.
CMOS دىن ئىچكى ساقلىغۇچ چاستوتىسىنى تەڭشەيدىغان تۈر:
SD-RAM Frep
بىر قىسىم ئاساسىي تاختىلاردا ئىچكى ساقلىغۇچ چاستوتىسىنى CPU چاستوتىسى، PCI چاستوتىسى  بىرلىكتە تەڭشىلىدۇ.
CMOS دىن ئىچكى ساقلىغۇچقا مۇناسىۋەتلىك تەڭشەش تۈرلىرى خاتا تەڭشىلىش سەۋەبىدىن يوقىرىقىدەك ئەھۋاللار كۆرۈلسە، ئوخشاشلا يوقىرىدا تونۇشتۇرۇلغان ئىككى خىل ئۇسۇل بويىچە ئەسلىگە كەلتۈرۈشكە بولىدۇ.
س:256MB ئىچكى ساقلىغۇچتىن ئىككى دانە سەپلەنگەن. بىراق 256MB ئىچكى ساقلىغۇچنىلا تەكشۈرىدۇ، نېمە ئۈچۈن؟
ج: 1. ئاساسىي تاختا BIOS نەشىرى تۆۋەن بولسا بەلگىلىك قىممەتتىن ئېشىپ كەتكەن ئىچكى ساقلىغۇچنى تونىمايدۇ ھەم  ئىشلىمەيدۇ. شۇڭا BIOS نەشىرىنى يوقىرى كۆتۈرۈڭ.
2. ئاساسىي تاختىدىكى ئۆزەكلەر گۇرۇپپىسى قوللىمىغان بولىشى مۇمكىن، ئاساسىي تاختىنىڭ يېڭى نەشىرىدىكى قوزغىتىش پروگراممىسىنى قاچىلاڭ.
س: ئىچكى ساقلىغۇچ قىزىپ كېتىشى سەۋەبىدىن سىستېمىنىڭ مۇقىم ئىشلىمەسلىكى ھەم قېتىۋېلىشى قانداق ئەھۋال؟
ج: ھاۋانىڭ ئىسسىپ كېتىشى، ئاساسىي ماشىنا قېپىنىڭ ئىسسىقلىق تارقىتىش ئۈنۈمىنىڭ ياخشى بولماسلىقى سەۋەبىدىن ئىچكى ساقلىغۇچ قىزىپ كەتسە، كومپيوتېر قېتىۋالىدۇ ياكى قوزغالمايدىغان ئەھۋال كۆرىلىدۇ.
ئىچكى ساقلىغۇچ زىچ ئورۇنلاشتۇرۇلغان توپلاشتۇرۇلغان يېرىم ئۆتكۈزگۈچ ماتېرىيالى بولۇپ، يېرىم ئۆتكۈزگۈچ ماتېرىيال بەلگىلىك تېمپراتورا دائىرىسىدە نورمال ئىشلەيدۇ، تېمپراتورا بۇ دائىرىدىن ئېشىپ كەتسە، سانلىق مەلۇمەتلارنى يەتكۈزۈش، ساقلاش، ئوقۇش جەريانىدا ئاسانلا خاتالىق كۆرۈلىدۇ.
1. ئاساسىي ماشىنا قېپى ئىچىگە يەنە بىر دانە شامالدۇرغۇچ ئورنىتىپ، سوۋۇتۇش ئىقتىدارىنى ياخشىلاڭ.
2. ھاۋا بەك ئىسسىپ كەتكەندە ئامال بار ھاۋا تەڭشىگۈچ ئىشلىتىڭ.
3. ئىچكى ساقلىغۇچقا ئىسسىقلىق تارقىتىش پلاستىنكىسى سەپلەڭ.
س: كومپيوتېرنى قوزغىتىپ ئۇزۇن ئۆتمەي قېتىۋالىدۇ، قايتا قوزغاتقاندا بىخەتەر ھالەتتە قوزغىلىدۇ، بۇ نېمە ئۈچۈن؟
ج: بۇ خىل كاشىلا ئىچكى ساقلىغۇچ ئەتراپىغا كۆپ مىقداردا چاڭ - توزان توپلىشىۋالغانلىق ھەم ئىچكى ساقلىغۇچ قىستۇرۇش ئوقۇرىنىڭ مېتال قىسمىنىڭ ئوكسىدلىنىشى سەۋەبىدىن بولغان. چوتكا ۋە قېرىنداش قەلەم ئۆچۈرگۈچ ئارقىلىق ئىچكى ساقلىغۇچنىڭ ماتېل قىسمىنى ۋە ئىچكى ساقلىغۇچ قىستۇرۇش ئوقۇرىنى تازىلاش كېرەك.
س: قاتتىق دىسكىغا قاچىلانغان مەشغۇلات سىستېمىسى ۋە يۇمشاق دېتاللارنى باشقا قاتتىق دىسكىغا ئۆز پىتىچە كۆچۈرگىلى بولامدۇ؟
ج: Ghost يۇمشاق دېتالى ئارقىلىق بولىدۇ. قاتتىق دىسكا سىغىمى ئوخشاش بولساdisk to disk بۇيرۇقى ئارقىلىق ئۆز پىتىچە كۆچۈرگىلى بولىدۇ. ئەگەر دىسكا سىغىمى ئوخشاش بولمىساpartition to partition بۇيرۇقى ئارقىلىق سىغىمى ئوخشاش بولغان رايون بويىچە كۆچۈرگىلى بولىدۇ.
س: CMOS نىڭ قاتتىق دىسكىنى تەكسۈرەلمەسلىكى قانداق ئەھۋال؟
ج: 1. قاتتىق دىسكا ۋىرۇسنىڭ بۇزغۇنچىلىقىغا ئۇچىرىغان ئەھۋالدا قاتتىق دىسكىنىڭ لوگىكىلىق قۇرۇلمىسى، يېتەكلەش سانلىق مەلۇماتلىرى، ھۆججەت تەقسىملەش جەدۋىلى ..قاتارلىقلار بۇزغۇنچىلىققا ئۇچرىغانلىق سەۋەبىدىن CMOS قاتتىق دىسكىنى تەكشۈرەلمىگەن. بۇنداق ئەھۋالدا قاتتىق دىسكىنى قايتىدىن رايونلارغا ئايرىپ فورماتلىسىڭىز بولىدۇ.
2. قاتتىق دىسكىدا فىزىكىلىق بۇزۇلۇش بولسىمۇ CMOS قاتتىق دىسكىنى تەكشۈرەلمەيدىغان ئەھۋال كۆرىلىدۇ. قاتتىق دىسكىنىڭ ئۇچۇر سىمى، توك مەنبە سىملىرىنى تەكشۈرۈپ كۆرۈڭ. ياخشى تىگىشمىگەن ئەھۋال بولسا قايتىدىن مۇقىملاشتۇرۇڭ. يەنىلا تەكشۈرۈلمىسە، قاتتىق دىسكىنىڭ ماگنىت بېشى ۋە كونترول توك يولىدا كاشىلا كۆرۈلگەن، كەسپى رېمونتخانىغا ئېلىپ بېرىپ ئوڭشىتىڭ. ئەگەر كومپيوتېردا مۇھىم سانلىق مەلۇماتلار بولسا يېڭىدىن بىر قاتتىق دىسكا سەپلەڭ، رېمونت قىلىنغان قاتتىق دىسكىنىڭ مۇقىملىقى ياخشى بولمايدۇ.
